ES6 features are used:

  • let(apply.js,line57,line70) and const(apply.js,line46-50):
    let provides block‑scoped variables so each loop iteration or block has its own value, which prevents bugs caused by variable leakage in ES5; const declares fixed references that cannot be reassigned, which is ideal for DOM elements or configuration objects like my division map.
  • NodeList.forEach(apply.js,line79)
    In ES5, querySelectorAll returns a NodeList that does not support forEach. ES6 extended NodeList so it behaves more like an array. I used forEach here to loop through all buttons and attach event handlers in a cleaner way than a traditional for loop.
  • Element.closest()(apply.js,line81,line86)
    closest() is an ES6/DOM Level 4 method that finds the nearest ancestor element matching a selector. In ES5, I would have needed a manual loop with parentNode. Here, I used closest('form') to directly find the form containing the button, which makes the code shorter and easier to read.
